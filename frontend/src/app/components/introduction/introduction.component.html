Comprendre le problème de l'arbre de Steiner

Imaginez que vous devez construire un réseau routier reliant trois villes. Votre objectif est de minimiser la longueur totale des routes tout en assurant la connexion de toutes les villes.

C'est l'essence du problème de l'arbre de Steiner, nommé d'après le mathématicien Jakob Steiner (1796-1863).

Applications réelles : télécommunications, conception de circuits, réseaux de pipelines

{{ slides[1].title }}

{{ slides[1].subtitle }}

La solution la plus évidente est de connecter directement chaque ville à toutes les autres, formant un graphe complet.

Cela fonctionne, mais c'est gaspilleur - nous construisons plus de routes que nécessaire !

Longueur totale : 640 km Inefficace

{{ slides[2].title }}

{{ slides[2].subtitle }}

Un arbre couvrant minimal (MST) connecte toutes les villes sans former de cycles, en utilisant les arêtes les plus courtes possibles.

C'est bien mieux ! Nous avons supprimé l'arête A-B car nous pouvons passer par A → C → B.

Longueur totale : 400 km Mieux (-37%)

Mais peut-on faire encore mieux ?

{{ slides[3].subtitle }}

L'arbre de Steiner ajoute un nouveau point de jonction (appelé point de Steiner) qui ne correspond à aucune ville.

En plaçant ce point stratégiquement, nous pouvons réduire encore plus la longueur totale des routes !

Longueur totale : 360 km Optimal (-44%)
Le point violet S est le point de Steiner - une jonction que nous avons ajoutée !

{{ slides[4].subtitle }}

Pour exactement 3 villes, le point de Steiner optimal est appelé le point de Fermat (ou point de Torricelli).

Il a une propriété remarquable : les trois angles sont exactement de 120°.

Propriété mathématique : ∠AFB = ∠BFC = ∠CFA = 120°
Ce point minimise la somme des distances vers les trois villes !
