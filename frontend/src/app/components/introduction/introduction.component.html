Étape 1 sur 5  ·  Le problème

Connecter des villes avec un minimum de routes

Un problème d'optimisation classique avec un impact réel

Imaginez construire un réseau routier reliant trois villes. Votre objectif : minimiser la longueur totale des routes tout en maintenant chaque ville connectée.

C'est le problème de l'arbre de Steiner, nommé d'après le mathématicien suisse Jakob Steiner (1796–1863). Il apparaît dans d'innombrables domaines de l'ingénierie.

Télécommunications
Minimiser la longueur de câbles dans les réseaux fibres optiques reliant les centres de données
Réseaux routiers
Concevoir des autoroutes et pipelines connectant des régions au coût minimal
Conception de circuits
Connecter des composants électroniques avec une longueur de trace minimale sur PCB
Biologie computationnelle
Reconstruire des arbres évolutifs (phylogénétique) à partir de séquences ADN
Le problème de l'arbre de Steiner est NP-difficile en général — aucun algorithme exact en temps polynomial n'est connu pour de grandes instances.
Étape 2 sur 5  ·  Approche naïve

Graphe complet — tout connecter

La solution évidente n'est pas la meilleure

L'idée la plus simple : construire une route directe entre chaque paire de villes. Cela crée un graphe complet où chaque ville peut atteindre toutes les autres.

Pour n villes, cela nécessite n(n−1)/2 arêtes. Avec 3 villes, c'est 3 routes — déjà redondant.

Longueur totale : 772 km Pire
1
Avec 3 villes : 3 arêtes, une entre chaque paire — toutes sont conservées.
2
Avec 10 villes : 45 arêtes — croissance exponentielle du coût.
3
De nombreuses routes sont redondantes — vous pourriez en supprimer et garder toutes les villes connectées.
Un graphe complet avec cycles est toujours sous-optimal. On peut toujours supprimer au moins une arête sans déconnecter de ville.
Étape 3 sur 5  ·  Approche MST

Arbre couvrant minimal

Supprimer les arêtes redondantes — garder seulement le nécessaire

Un Arbre Couvrant connecte toutes les n villes en utilisant exactement n−1 arêtes sans cycles. La version Minimale choisit le plus court ensemble possible.

Les algorithmes classiques résolvent le MST en temps O(E log V) — très efficace même pour des milliers de villes.

Longueur totale : 472 km Mieux (−39%)
1
Kruskal's algorithm: sort all edges by length, add shortest edge that doesn't create a cycle.
2
Prim's algorithm: grow the tree greedily from any city, always adding the nearest unvisited city.
3
Both give the same optimal result — the MST is unique when all edges have different lengths.

Can we do even better by adding new junction points?

Étape 4 sur 5  ·  Arbre de Steiner

Ajouter des points de jonction pour réduire la distance

Le point de Steiner n'est pas une ville — c'est une jonction calculée

L'élément clé de l'arbre de Steiner : nous pouvons ajouter de nouveaux points intermédiaires (appelés points de Steiner) qui ne sont pas des villes d'origine. Les routes se croisent à ces jonctions, réduisant la longueur totale.

Un arbre de Steiner sur n terminaux peut avoir au plus n−2 points de Steiner. Pour 3 villes, cela signifie au plus 1 jonction supplémentaire.

Longueur totale : 439 km Optimal (−7%)
Complexité
NP-difficile pour ≥3 terminaux. Approximation à un rapport de 1.39 (Robins–Zelikovsky)
Économies
Toujours ≤ longueur du MST. Économie maximale : ~13,4% pour un triangle équilatéral
Le point S violet n'est pas une ville — c'est une jonction virtuelle que nous avons introduite pour réduire la longueur totale du réseau.
Étape 5 sur 5  ·  Le point de Fermat

La règle des 120° pour 3 villes

Une belle propriété géométrique nous donne la solution exacte

Pour exactement 3 points terminaux, le point de Steiner optimal est connu sous le nom de point de Fermat–Torricelli, découvert indépendamment par Pierre de Fermat et Evangelista Torricelli au XVIIe siècle.

Sa propriété définissante : chaque angle entre deux routes est exactement de 120°. C'est le point unique qui minimise la distance totale aux trois villes.

Propriété mathématique ∠AFB = ∠BFC = ∠CFA = 120°
1
Construire un triangle équilatéral sur chaque côté du triangle d'origine.
2
Tracer des lignes depuis chaque nouveau sommet extérieur vers la ville d'origine opposée.
3
Les trois lignes se croisent au point de Fermat — c'est notre jonction optimale.
Notre solveur utilise l'algorithme itératif de Weiszfeld pour trouver ce point numériquement — convergeant vers le point de Fermat exact en millisecondes.
